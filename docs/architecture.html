Data Flow — Filter Change to UI Update

1
User changes a filter in FilterBar

FilterBar.vue mutates one of the singleton refs in useFilters.js — e.g. selectedLocation.value = 'Tokyo'

2
Watch listener fires in active view

Each view watches [selectedPeriod, selectedLocation, selectedCategory, selectedStatus]. Any change triggers loadData().

3
api.js builds query params and fires axios GET

getCurrentFilters() maps composable refs → API param names. Non-'all' values appended via URLSearchParams. Example: GET /api/orders?warehouse=Tokyo&month=2025-03

4
FastAPI route handler filters in-memory data

apply_filters() iterates the loaded list with case-insensitive matching. filter_by_month() expands quarters (e.g. Q1-2025 → ['2025-01','2025-02','2025-03']) and substring-matches order_date.

5
Pydantic validates & serializes response

Filtered list is validated against response model (e.g. List[Order]) and returned as JSON.

6
Vue component stores raw data in ref

e.g. orders.value = response.data. Source ref updated, triggering downstream computed properties.

7
Computed properties re-derive UI data

Summary stats, chart data, filtered lists all recalculate automatically from the new ref value. Vue re-renders only the changed DOM nodes.

Key Architectural Patterns

Singleton Composables

Filter state (useFilters.js) lives at module level — refs are shared across all components that import the composable. No Vuex or Pinia needed.

Watch-Triggered Reload

Each view watch()es all four filter refs. Any filter change fires loadData() automatically, keeping UI in sync without manual event wiring.

Ref + Computed Separation

Raw API responses stored in plain ref()s. All derived values (chart data, summaries, filtered lists) live in computed(), cached until their deps change.

Pydantic Response Validation

All FastAPI routes declare typed return models. Ensures structural consistency between JSON files and what the frontend receives; catches data issues at startup.

i18n via Composable

useI18n.js provides t(key), currency conversion (USD/JPY), and locale-aware name translation. Locale persisted to localStorage.

Centralized API Client

All HTTP calls live in api.js. Components never call axios directly. Filter mapping (composable names → query param names) is isolated here.
